# 弹窗控制设计

## 目标

扩展弹窗交互模型，使卡片弹窗支持真正随机的屏幕内位置，弹幕弹窗在完全离屏前保持可见，弹窗可见时可通过可配置快捷键触发操作，并让 IPA 文本在弹幕头部保持可读。

## 范围

- 为卡片弹窗添加真正随机位置。
- 修复弹幕退出时机，让弹窗完整宽度移出左侧屏幕边界后才关闭。
- 添加 `known`、`unknown` 和 `dismiss` 三个可配置快捷键。
- 限制新增快捷键只作用于当前可见的卡片或弹幕弹窗。
- 调整弹幕布局，使 IPA label 即使在摘要和动作按钮存在时仍保持可见。
- 验证后构建新的 Windows 包。

## 架构

现有弹窗实现已经各自负责定位和动作接线，因此变更应保持增量。controller 仍作为可见弹窗动作的协调点；设置持久化继续通过 `AppSettings` 和 `SettingsStore` 流动；`GlobalHotkeyService` 继续负责注册和分发全局快捷键。

唯一值得抽取的共享逻辑，是卡片和弹幕代码会明显重复的小型定位或弹窗动作 helper。这样既保持当前结构，又避免不必要重构。

## 设计决策

### 卡片随机位置

`OverlayPosition.RANDOM` 应同时适用于卡片和弹幕模式。对卡片而言，每次 `show_popup()` 都应在当前屏幕可用几何范围内生成一个新的随机坐标，并 clamp 到完整弹窗可见。

### 弹幕生命周期

弹幕动画应从右侧边界外开始，移动到左侧边界外完全离屏。关闭仍绑定动画完成，但结束位置必须考虑弹窗宽度，避免内容仍在屏幕上时动画已经结束。

### 仅可见弹窗快捷键

设置和热键注册新增三个动作：

- `known`
- `unknown`
- `dismiss`

这些动作像现有快捷键一样全局注册，但 controller 在没有当前可见弹窗或没有 active entry 时必须 no-op。同一个动作应同时适用于卡片和弹幕模式。

### IPA 布局

当前弹幕头部可能让摘要和动作按钮占据整行宽度，导致 IPA 隐藏。头部应重新平衡：单词和 IPA 位于稳定文本列，动作按钮位于独立控制区。IPA label 应在省略前保留可见空间。

## 测试

- 添加回归测试，覆盖新增热键字段的设置规范化与持久化。
- 添加卡片随机位置测试，确保弹窗保持在屏幕边界内。
- 添加弹幕运动测试，证明结束位置已完全离屏。
- 添加 controller 或热键动作测试，证明新增可见弹窗动作在没有可见弹窗时 no-op，在存在弹窗时正确分发。
- 实现后运行完整 `pytest` 测试套件。

## 风险

- 弹幕动画 bug 对几何敏感，测试应直接断言坐标，而不是依赖人工观察。
- 仅可见快捷键需要避免在弹窗关闭后继续作用于陈旧弹窗状态。
- 布局修复必须保留当前按钮行为和弹幕模式下的 hover/pause 行为。
