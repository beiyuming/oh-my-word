# 弹窗自动朗读与延迟朗读设计

## 目标

为单词弹窗新增两个用户可配置能力：

- 是否在单词弹出后自动朗读
- 自动朗读前等待多少秒，让用户先看一眼单词

该能力适用于卡片和弹幕两种展示模式，并继续复用现有 `PronunciationService`、朗读文本拼装和朗读成功记录逻辑。

## 设计决策

1. 设置字段新增到 `AppSettings`，默认关闭自动朗读。
2. 延迟单位使用秒，允许小数，默认值为 `1.0` 秒。
3. 自动朗读调度由 `AppController` 负责，不下沉到 `CardPopup` / `BarragePopup`。
4. 自动朗读触发时仍走现有 `pronounce_current_word()` / `pronounce_text()` 链路，不复制一套 TTS 逻辑。
5. 若用户在自动朗读前手动点了朗读或关闭了弹窗，待触发的自动朗读应取消，避免重复播放或读到旧词。

## 设置契约

新增字段：

- `auto_pronounce_on_popup: bool = False`
- `auto_pronounce_delay_seconds: float = 1.0`

规范化规则：

- `auto_pronounce_on_popup` 仅接受布尔值，否则回退默认值
- `auto_pronounce_delay_seconds` 允许 `0.0` 到 `10.0`
- 非数字、负数回退默认值
- 超过上限夹到 `10.0`
- 存储时保留两位小数

## UI 设计

放在设置页“发音”分类，与“朗读内容”“发音引擎”相邻：

- 复选框：`单词弹出后自动朗读`
- 数值框：`自动朗读延迟`

行为：

- 复选框关闭时，延迟设置仍保留但不生效
- 数值框后缀显示 `秒`

## 控制流

### 单词弹出

`AppController._show_word()` 在展示弹窗后：

1. 先取消上一次尚未触发的自动朗读
2. 根据当前设置判断是否应调度自动朗读
3. 如启用，则启动单次定时器

### 自动朗读触发

定时器触发时必须再次确认：

- 当前单词仍是原来的单词
- 当前仍有可见弹窗
- 没有开启静音
- 自动朗读设置仍然启用

全部满足时再调用 `pronounce_current_word()`

### 自动取消条件

以下场景必须取消待触发的自动朗读：

- 新单词顶掉旧单词
- 弹窗被关闭、dismiss、snooze、review
- 用户手动触发朗读
- 设置被改为关闭自动朗读

## 与现有 TTS 的关系

- `system_qt` 路径：保持现有初始化与失败提示逻辑
- `voxcpm_local` 路径：保持现有自动启动、异步播放和 `last_pronounced_at` 记录逻辑
- 自动朗读只是新增触发源，不改变 provider 语义

## 验证

- 设置规范化与持久化测试
- 设置窗口 round-trip 测试
- controller 调度/取消自动朗读测试
- controller 在旧词失效或弹窗关闭时不应误触发朗读测试

## 不在本次范围内

- 为自动朗读新增单独快捷键
- 为不同展示模式设置不同延迟
- 为自动朗读增加仅首词/仅新词等高级策略
